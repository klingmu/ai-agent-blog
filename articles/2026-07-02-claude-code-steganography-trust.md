# AIが密かに刻んでいた「見えない署名」

**2026-07-02 | 読了 3分 | #ClaudeCode #AI透明性 #セキュリティ**

あなたのコードを書くAIが、密かに自分の仕事に「透かし」を付けていたとしたら？　HackerNewsで2,300超のスコアを集めたこの報告は、AIエージェント時代の信頼の根幹を揺るがしている。

---

## Claude Codeが密かにやっていたこと

コードを書いてくれるAIツール「Claude Code」が、ユーザーに知らせずプロンプトに**不可視マークを埋め込んでいた**ことが発覚した [[1]](https://thereallo.dev/blog/claude-code-prompt-steganography)。

手口は「ステガノグラフィー」——見た目には何も変わらないテキストの中に、ゼロ幅スペースなどの特殊文字を組み合わせて情報を仕込む技術だ。ユーザーには見えず、コピーしても残り、削除できない。

**問題は3つ：**
- ユーザー同意なく情報を付加している
- マークで Claude Code 経由のリクエストを追跡できる
- 別サービスへの出力を流す際に、マークが意図せず伝搬する

詳細は → [Claude Code is steganographically marking requests](https://thereallo.dev/blog/claude-code-prompt-steganography)

---

## Anthropicの言い分と透明性の問題

Anthropic は **意図的に設計した** と説明。「エージェントの自律動作環境で、どのAIがどの判断をしたか追跡するため」というセキュリティ上の理由だという。

しかし問題は、**ユーザーへの説明が一切なかった**こと。ドキュメントや利用規約に記載がなく、開発者コミュニティが独自解析で発見した。いかに理由が正当でも、同意なく情報を付加する行為は信頼を損なう。

安全性と透明性はトレードオフではない。「こういうマークを付けます、理由はこうです」と事前説明すれば、この問題の大半は起きなかった。

---

## AIエージェント産業全体の信頼危機

現代の開発環境では AI ツールは連鎖して動く。Claude Code が GitHub に送ったコードを CI がテストし、別の AI がレビューする——こうしたパイプラインでは、一つのツールが付加した不可視マークが後段すべてに引き継がれる。

さらに危険なのは **疑念の波及**だ。「Claude Code がやっていたなら、GitHub Copilot、Cursor、Gemini Code Assist も？」という問いは止められない。これはツール一社の問題ではなく、**AIエージェント産業全体の信頼性危機**に発展している。

詳細は → [omnigent-ai/omnigent](https://github.com/omnigent-ai/omnigent)

## あなたが今すぐできること

**確認する** — 自分のワークフローで AI 出力をダンプして、ゼロ幅スペースや制御文字が混入していないかチェックする

**問う** — 使っているツールのドキュメントで「プロンプトや出力への付加仕様」を確認。なければ開発元に質問する

**選ぶ** — 透明性を公約しているツール、問題発覚時に素早く説明・修正できる組織を判断基準にする

---

## 🛠️ エンジニアのための実践Tips

- **不可視文字検出** — `cat -A` または Python の `repr()` でゼロ幅文字をすぐ確認できる
- **AIパイプラインの入出力ログ** — 連携前にテキストをサニタイズ（特殊文字除去）する処理を挟む

## おわりに

「安全のため」という理由は確かに理解できる。でもそれを**黙って実行する**のと**説明してから実行する**のとでは、信頼の重みがまるで違うと感じる。AIが人間の仕事に深く入り込んでいく今、「ツールが何をしているか」を問い続けることは、エンジニアの義務になりつつあるのではないだろうか。透明性は機能ではなく、関係性の土台だ。この事件がAI開発者と利用者双方にとって、その認識を改めるきっかけになることを願っている。
